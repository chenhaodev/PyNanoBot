# 路线图（草案）

以下为方向性规划，优先级会随社区与维护资源调整；详细设计以 Issue / ADR 为准。

## 近期

- [ ] **上游同步节奏**：固定周期（如每 2–4 周）或按上游 release 合并，并更新 `upstream.lock`。
- [ ] **文档**：本地推理（Ollama / OpenAI-compatible / llama.cpp 后端）的推荐配置矩阵（上下文长度、工具调用、多模态开关）。
- [ ] **测试**：对「小上下文窗口」场景的回归（提醒注入、历史裁剪、compactor 可选路径）。

## 中期：本地与中小模型

- [ ] **GGUF / 本地权重**：通过现有 Provider 抽象对接常见本地服务（不重复造推理引擎），统一文档与 env 模板。
- [ ] **默认策略**：资源受限时的 tool 结果截断、会话长度、reminder 频率的预设 profile（`local`, `edge`, `cloud`）。
- [ ] **可选依赖**：将重型通道或可选栈放入 `optional-dependencies`，减小「最小安装」体积。

## 长期：生态

- [ ] **技能与模板**：与上游 skills 兼容，增加面向本地开发者的模板仓库或索引说明。
- [ ] **可观测性**：结构化日志、可选 OpenTelemetry，便于自建部署。

欢迎在本仓库开 Issue 讨论优先级与拆分任务。
